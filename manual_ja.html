1. 概要

TDN Geo-Rotating Arpeggiator はキネティック音楽楽器です。キーを押して音を出すのではなく、動きによって音がトリガーされます。カーソル（または指）がキャンバス上を移動した距離が、次の音が鳴るタイミングを決定します。

キャンバスには布のようなメッシュが表示され、ドラッグ操作にリアルな物理演算で反応します — 伸び、しわが寄り、離すとバネのように戻ります。この物理的な変形は視覚的なだけでなく、ピッチ、ベロシティ、フィルター、タイミングをリアルタイムで直接変調します。

この楽器はひとつの理論的原理 T(D,N) に基づいています。T(D,N) とは、単一のパラメータ N によって、非ダイアトニック音（黒鍵）のタイミングとピッチを同時に変換する二重変換です。

N = 1.0 のとき、楽器は標準の12平均律と同じ動作をします。N が 0 に近づくにつれ、非ダイアトニック音（黒鍵）は音価が短くなり、ピッチが隣のダイアトニック音（白鍵）に近づきます。

2. 演奏方法

自動再生（Auto Play）

サイドバーの ▶ PLAY ボタンをクリックします。カーソルが幾何学パターンに沿って自動的に動き、音楽を生成します。横のドロップダウンでパターンの形を選んでください。Speed スライダーでカーソルの速度を調整します。

手動演奏（フィールドドラッグ）

キャンバス上でクリックしてドラッグします。布のメッシュが指やカーソルの下で変形します。自動再生カーソルが動くと音がトリガーされ、同時にドラッグがフィールドドラッグシステムを通じて音を変調します — ピッチ、ベロシティ、フィルター、スウィングをリアルタイムで変化させます。マウスを離すと布がバネのように戻ります。

4. T(D,N) — 核心的な原理

T(D,N) は「パラメータ N によるテンポラル・ダイアトニック変換」を意味します。この楽器の定義的特徴であり、単一のスライダーが音楽の2つの次元を同時にコントロールします。

ダイアトニック参照集合 D

D は標準的なピアノの白鍵に対応するピッチクラスの集合です：

D = { 0, 2, 4, 5, 7, 9, 11 } → C, D, E, F, G, A, B

これは調性的なスケールではなく、位相的な参照集合です。どの音がダイアトニック（白鍵）で、どの音が非ダイアトニック（黒鍵）かを定義します。

パラメータ N

N は 0 から 1 の範囲で、非ダイアトニック音に対して2つのことを同時に行います：

次元ダイアトニック音（白鍵）非ダイアトニック音（黒鍵）
タイミング（トリガー閾値）基本閾値閾値 × N（短縮＝シャッフル）
ピッチ（セント補正）0¢+ (1 − N) × 100¢
例 — N = 0.5、Iceface チューニング：
C#（非ダイアトニック）は通常の半分のトリガー間隔で発音されます — 隣の白鍵に「引き寄せられる」ように早くトリガーされます。同時にピッチは +50¢ だけ D に向かってシフトされます。Iceface チューニングも C# を平均律の位置から +50¢ ずらしています。

両方の変換が同じ値に収束します — ピッチとタイミングの圧縮が、単一の N によって統一されています。

Z-Depth

ベロシティとフィルターカットオフの深さを制御します。N はベロシティやカットオフに影響しません — それらは Z-Depth のみで駆動されます。Z-Depth = 0 では全ての音が均等な音量と明るさで鳴ります。Z-Depth = 100% では音が背景に押しやられ、静かで暗くなります — 3D空間の遠い層のように。

スウィング（Swing）

ステップのタイミングにスウィング（揺れ）を適用します。50%（中央）では全ステップが均等。スウィングを上げると偶数ステップが短く、奇数ステップが長くなり、シャッフルやグルーヴ感が生まれます。フィールドドラッグでリアルタイム変調も可能です。

T(D,N) モード vs T(D,N) LEAP モード

T(D,N) — オリジナルの変換。ステップのタイミングは現在の音の属性（D に属するかどうか）によって決まります。

T(D,N) LEAP — 拡張モード。ステップのタイミングは現在の音と前の音の距離（半音数）によって決まります。大きな跳躍は長い音価を、隣接音は短い音価を得ます。

T(D,N) モード: ダイアトニック音 → 基本閾値（固定） 非ダイアトニック音 → 閾値 × N（短い間隔 ＝ シャッフル） T(D,N) LEAP モード: interval = |現在の音 − 前の音| （半音数） stretch = 1.0 + (interval − 1) / 10 → 1度: 1.0×、11度: 2.0× threshold = 基本閾値 × (1 + (stretch−1) × N)

LEAP モードでは N がストレッチの強度をスケールします。N = 0 では全ステップが均等。N = 1.0 では大きな跳躍が自然に呼吸し、順次進行は素早く流れます。

音源として使用するピッチクラスを定義します。選択したスケール、Base ノート、オクターブ数の組み合わせが、キネティックエンジンがステップスルーする音列全体を決定します。

ICEFACE 系チューニングを選択すると、N が自動的に 0.5 に設定され、スケールが Chromatic に切り替わり、完全な T(D,N) 変換が有効になります。非 Iceface 系チューニングでは N が 1.0 にリセットされ、スケールが Major に戻ります。

ベースノート（Base）

生成されるスケールの最低音（C2〜C6の範囲）。音列はこの音から上方向に、指定したオクターブ数だけ広がります。

オクターブ数（Octaves）

音列がカバーするオクターブ数（1〜4）。広いほど音域が広がり、旋律の多様性が増します。

6. シンセサイザー

波形（Wave）

Sine（サイン波）は最も純粋な音色。Triangle（三角波）はサイン波に近いが少し明るい。Square（矩形波）は奇数倍音が強く中空でリード的な音色。Sawtooth（ノコギリ波）は全倍音を含む最もリッチな音色 — アシッドベースの基礎。

カットオフ（Cutoff）

ローパスフィルターのカットオフ周波数。低くすると音が暗く丸くなり、高くすると明るくオープンな音色になります。フィールドドラッグでリアルタイム変調が可能です。

レゾナンス（Reso）

カットオフ周波数付近の強調。高い値にすると特徴的な「ワウ」ピーク — アシッドベースのキャラクターが生まれます。

リバーブ（Reverb）

コンボリューション・リバーブのセンド量。シンセ声部に空間的な奥行きと余韻を加えます。0% では完全にドライ。高い値にすると広がりのある没入的なアンビエンスが生まれます。

グライド（Glide）

音符間のピッチ遷移をなめらかにするポルタメント。0 では即座に音程が変わります。増やすと前の音から新しい音へスライドします。

オクターブ（Octave）

音列全体をオクターブ単位でシフト（−2〜+2）。マイナス値にするとベース音域になります。

音量（Volume）

シンセサイザー声部の出力レベル。

7. ドラム

ドラムエンジンは Web Audio API を使って3つの声部をゼロから合成します。サンプルは一切使用していません。

声部合成方式
K — キックピッチスウィープ付きサイン波（160Hz → 45Hz）＋クリックトランジェント。0.4秒ディケイ。
S — スネアサイン波ボディ＋ノイズ＋スナップオシレーター＋アタックトランジェント。0.35秒ディケイ。
H — ハイハットフィルターされたホワイトノイズ、高速指数減衰。0.08秒。

ドラムステップはノートトリガーと同期して進みます。キネティックエンジンが音を鳴らすたびに、ドラムシーケンサーが1ステップ進みます。フィールドドラッグはドラムのベロシティとフィルターをリアルタイムで変調します。

音量（Volume）

全ドラム声部のマスターレベル。

フィルター（Filter）

ドラムバス専用のローパスフィルター。シンセサイザーの音色とは独立して調整できます。

8. 方向（Direction）

モード動作
UP（上昇）昇順に順番に進む。最高音から最低音に折り返す。
DOWN（下降）降順に順番に進む。最低音から最高音に折り返す。
BOUNCE（反転）最高音まで上がったら降り始め、最低音まで下がったら昇り始める。
RANDOM（ランダム）各トリガーで音列からランダムに選ぶ。T(D,N) LEAP と組み合わせると音程比例タイミングが働く。

9. キネティック・パラメータ

ステップ（Step）

次の音をトリガーするために必要な基本距離（キャンバスの幅を1とした単位）。小さい値にすると同じ速度でもより速く音が鳴ります。この値は T(D,N) によって非ダイアトニック音に対して修正されます。

スピード（Speed）

Auto Play 中のみ有効。カーソルが幾何学パターン上を移動する速度を制御します。上げると密なリズムに、下げるとゆったりとした間の多い演奏になります。

10. フィールドドラッグ — 布の物理演算

キャンバスには織り布のメッシュが表示されます。クリックしてドラッグすると、布がリアルな物理演算で変形します — 伸び、しわが寄り、離すとバネのように戻ります。これが Geo-Rotating Arpeggiator の定義的なインタラクションモデルです。

フィールドドラッグは視覚効果だけではありません。布の物理的状態が音楽をリアルタイムで直接変調します。布を引っ張ると楽器の音が変わります — 自動再生中でも有効です。

仕組み

キャンバス上でプレスしてドラッグすると、システムは以下を追跡します：

  • アンカーポイント — 最初にプレスした位置
  • 布の変位 — 布がどこまで引っ張られたか（バネ物理＋ダンピング付き）
  • ストレイン — 生のドラッグと布の現在位置の間の張力
  • スピード — ドラッグの速度
  • エネルギー — アクティブなドラッグ中に上昇し、リリース時に滑らかに減衰する複合値

変調対象

パラメータソース効果
X Warp水平方向の布の変位スウィングタイミングのオフセットを変調
Tone垂直方向の布の変位シンセフィルター・カットオフとドラムフィルターを変調。上にドラッグ＝明るく、下＝暗く。
Depthエネルギー＋ストレインZ-Depth リフトを追加 — ベロシティと空間的奥行きに影響
Energy全ドラッグ力の複合値全体的な変調強度。ベロシティブースト、キネティック加速、ビジュアルエフェクトを駆動。

サイドバーの Field Drag 表示は、X Warp、Tone、Depth、Energy の現在値をリアルタイムで表示します。

布のバネ動作

布はバネ-ダンパーモデルを使用しています。引っ張って離しても即座にはスナップバックしません — 振動してから落ち着きます。これにより、リリース後も音の変調が短時間続き、有機的で減衰する表現が生まれます。ドラッグが重いほどバネ定数が遅くなり、布のような重量感のある手触りになります。

11. ドラムパターングリッド

画面下部のドラムパターンエディタには、K（キック）、S（スネア）、H（ハイハット）の3行が表示されます。

編集方法

任意のセルをクリックするとオン／オフが切り替わります。アクティブなセルは色で強調表示されます（キック＝ピンク、スネア＝ブルー、ハイハット＝ゴールド）。現在再生中のステップはブルーの枠で示されます。

操作ボタン

  • パターンドロップダウン — 内蔵パターンを読み込む。
  • RANDOM — ランダムにパターンを生成。キック 30%、スネア 20%、ハイハット 40% の確率でオンになります。

パターン長

パターンの長さはプリセットによって異なります（12ステップまたは16ステップ）。ドラムシーケンサーは音列の長さとは独立してループするため、長さが異なるとポリリズム的な関係が生まれます。

12. BPM表示

ヘッダーバーに ~ 120 TEMPO の形式で推定テンポが表示されます。これは直近のノートトリガー間の平均間隔から計算されます。この楽器のタイミングはクロックベースではなく空間ベースのため、BPM はカーソル速度、フィールドドラッグのエネルギー、ステップサイズに応じて変動します。固定メトロノームではなく参考値です。
