W H A T   I S   R E L A T I O N A L ?

What does "Relational" mean?

Conventional: Absolute

Each note has a fixed value.

Pitch: C# = 100 cents (always fixed)
Rhythm: all notes spaced equally

No structural relationship exists between notes.

12-TET / conventional arpeggiators

Iceface: Relational

Each note is defined by its relationship to neighboring structural notes.

Pitch: black key = defined by distance N to right-neighbor white key
Rhythm: black key = defined by interval ratio N to white key

A single parameter N controls the behavior of all non-diatonic notes.

Core principle: In music, "white keys" serve as structural reference points (diatonic reference), and "black keys" are defined by their relationship (distance N) to those reference points. This principle can be applied not only to pitch but to any musical parameter.

R E L A T I O N A L - T O N A L

I(D,N) — Iceface Tuning

Black key pitch = right-neighbor white key − N cents

The first form of the Relational principle applied to pitch space. Implemented in Iceface Toy One. White key pitches are fixed; only black keys move with N.

N = 0¢

Diatonic Pure

Black key = same as right-neighbor white key (absorbed → vanishes)

Only C D E F G A B remain

N = 50¢

Iceface Tuning

Black keys pulled 50¢ toward right-neighbor white key

Unique microtonal tuning

N = 100¢

12-TET (Conventional)

Black key = standard semitone

All notes equidistant

R E L A T I O N A L - T E M P O R A L

T(D,N) — Dual Transformation

The transformation that I(D,N) performs in pitch space, now simultaneously transcribed into time space. Realized in IcefaceTDN's KINETIC mode.

Time axis:   black key step interval = stepSize × N
Pitch axis:   black key offset = +(1 − N) × 100¢

N = 0

Diatonic Pure

Time: step = 0 → non-D notes completely skipped

Pitch: offset = +100¢ → same as right-neighbor white key (vanishes)

C D E F G A B play at even intervals

N = 0.5

Iceface Asymmetric

Time: step = step × 0.5 → triggers at half the white-key interval

Pitch: offset = +50¢ → pulled 50¢ toward right-neighbor white key

Complete spatiotemporal reproduction of I(D,N)

N = 1.0

Equal Interval (Conventional)

Time: step = step × 1.0 → same interval as white keys

Pitch: offset = 0¢ → 12-TET unchanged

Conventional chromatic arpeggiator

W H Y   D U A L ?

Why transform pitch as well as time?

Temporal-only (Incomplete)

Changing only the time interval by N:

At N = 0.5:
C# triggers at half the interval of C,
but pitch remains 12-TET (100¢)

Problem: Symmetry with I(D,N) breaks.
Iceface Tuning's concept is "pulling black keys toward white keys,"
but temporal-only pulls in time while leaving pitch distant.

Dual Transformation (Complete)

Changing time + pitch simultaneously by N:

At N = 0.5:
C# triggers at half the interval of C,
and pitch also shifts 50¢ toward D

Result: Perfect symmetry with I(D,N) is achieved.
Black keys are "pulled toward white keys" in both time and pitch.
At N = 0, both axes vanish → complete consistency.

I(D,N): pitch = right-neighbor white key − N¢
T(D,N): step = step × N,   pitch += (1−N) × 100¢

→ As N approaches 0, black keys are absorbed into white keys in both time and pitch.

I M P L E M E N T A T I O N

PlaybackEngine.tickKinetic() Implementation

Time Axis Transformation

// Diatonic check for current note
let note = noteArray[kineticSeqIndex]
let isDia = D.contains(note % 12)

// Determine step interval
if isDia:
  step = kStep         // standard
else if N ≈ 0:
  advanceIndex()     // skip
else:
  step = kStep × N     // scaled by N

// Trigger when distance exceeds step
if accum >= step → trigger

Pitch Axis Transformation

// Apply pitch offset at trigger time
let pc = note % 12
let isDia = D.contains(pc)

let cents =
  isDia ? 0.0          // white key: unchanged
  : (1−N) × 100.0    // black key: shifted

// Internal synth: direct microtuning
audio.playNote(note, cents)

// MIDI output: expressed via pitch bend
midi.pitchBend(8192 + cents × 40.96)

Note: Because current-note semantics are used, the same rules apply without additional peek logic even in RANDOM direction mode.
